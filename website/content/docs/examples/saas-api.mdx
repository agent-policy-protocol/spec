# SaaS API Platform

A policy template for API-first SaaS platforms that need open documentation access while protecting API endpoints and blocking admin/billing areas.

## Use Case

- Allow agents to freely read API documentation
- Gate API access to verified, allowlisted agents with high rate limits
- Block admin and billing endpoints entirely
- Support tool invocation for MCP-compatible agents

## How It Works

### Open Documentation

The `/docs/**` path allows full access including extraction (500 requests/hour). AI agents can learn your API by reading docs, which improves integration quality.

### Verified API Access

API endpoints require verification and are restricted to allowlisted agents. The `tool_invoke` action enables MCP-compatible agents to interact with your API as tools.

### Admin/Billing Lockdown

Admin and billing endpoints use `agentDenylist: ["*"]` to block all agents, including those that might otherwise be on an allowlist.

### Health Check

The `/health` endpoint is open without verification — useful for monitoring agents.
