# WordPress Default

A drop-in policy template for WordPress sites with sensible defaults covering common WordPress paths, REST API, and admin areas.

## Use Case

- Allow reading and indexing of public content
- Block admin login, XML-RPC, and wp-admin
- Allow REST API access with rate limits
- Gate WooCommerce API behind verification
- Support RSS feed indexing

## Full Policy

## How It Works

### Public Content

By default, agents can read, index, and summarize all public pages and posts. The 120 requests/hour rate limit prevents aggressive scraping.

### WordPress Admin Protection

Three critical WordPress paths are blocked:

- `/wp-admin/**` — The admin dashboard
- `/wp-login.php` — The login page
- `/xmlrpc.php` — The XML-RPC endpoint (common attack vector)

### REST API Access

The WordPress REST API (`/wp-json/wp/v2/**`) is open with 300 requests/hour, enabling agents to access structured post and page data.

### WooCommerce Protection

The WooCommerce API (`/wp-json/wc/v3/**`) requires verification and has stricter limits (100/hour), protecting e-commerce operations like order data.

### Media Files

Uploads at `/wp-content/uploads/**` are readable and indexable without verification.

### Deployment

1. Copy the JSON above to `agent-policy.json`
2. Upload to your WordPress root as `.well-known/agent-policy.json`
3. Alternatively, use an `.htaccess` rewrite:

```apache
RewriteRule ^\.well-known/agent-policy\.json$ /agent-policy.json [L]
```

### When to Use This Template

Use this as a starting point for any WordPress site. Customize the `pathPolicies` to match your specific plugin paths and requirements.
