# News Publisher

A policy template for news and media websites that need to protect premium content while allowing public articles to be indexed and summarized.

## Use Case

- Allow AI agents to read and index public articles
- Restrict premium/paywalled content to verified partners only
- Block content extraction to protect intellectual property
- Provide high rate limits for the archive, lower for premium
- Enable MCP and A2A interoperability

## Key Features

| Feature             | Configuration                                           |
| ------------------- | ------------------------------------------------------- |
| **Default Access**  | Read and index allowed, extraction blocked              |
| **Premium Content** | Read-only, verified agents only, allowlisted partners   |
| **Public Content**  | Read, index, summarize with generous rate limits        |
| **Archive**         | Fully accessible for reading, indexing, and summarizing |
| **Admin**           | Fully blocked                                           |
| **API**             | Accessible with verification                            |
| **Verification**    | DID and Verifiable Credentials                          |
| **Rate Limits**     | 60/hour default, 200/hour public, 20/hour premium       |

## How It Works

### Default Policy

All agents can read and index content by default with 60 requests/hour. Extraction, automated purchases, and form submissions are blocked.

### Premium Content Protection

The `/premium/*` path restricts access to read-only for verified agents on the allowlist. This protects paywalled articles from unauthorized summarization or data extraction.

### Public Content

The `/public/*` path allows generous access (200 requests/hour) with reading, indexing, and summarizing enabled — ideal for free-tier articles.

### Partner Verification

The `agentAllowlist` on premium content ensures only trusted AI agents (Perplexity, Gemini, SearchGPT) can access paywalled material after verification.
